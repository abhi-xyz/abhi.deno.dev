In the world of web development, choosing the right framework can make a significant difference in your productivity and the performance of your application. Rust, known for its performance and safety, has been making waves in various domains, and web development is no exception. Among the Rust ecosystem's promising offerings is the Yew framework. If you're looking to build fast, reliable, and modern web applications, Yew might be the tool you've been searching for. In this blog post, we'll dive into what Yew is, its key features, and why you should consider using it for your next project.

1. What is Yew?

Yew is a modern Rust framework for creating multi-threaded front-end web applications with WebAssembly (Wasm). Inspired by frameworks like React and Elm, Yew offers a powerful and ergonomic way to build interactive web applications. It leverages Rust's safety and performance to provide a robust development experience and high-performance runtime for your applications. Key Features of Yew

  1. Component-Based Architecture

Yew embraces a component-based architecture similar to React. Components are the building blocks of your application, encapsulating logic, state, and UI. This modular approach makes it easy to manage and reuse code, leading to cleaner and more maintainable projects.

  1. Zero-Cost Abstractions

Rust is renowned for its zero-cost abstractions, and Yew is no exception. It provides abstractions that allow you to write high-level code without sacrificing performance. Yew's efficient rendering model ensures that your application remains responsive, even as it scales.

  1. Asynchronous Programming

Yew integrates seamlessly with Rust's asynchronous programming model. You can use Rust's async/await syntax to handle asynchronous tasks, such as making API requests or performing background computations, without blocking the main thread.

  1. Type Safety

One of Rust's greatest strengths is its type system, which helps catch errors at compile time. Yew leverages this feature to ensure that your UI components are type-safe, reducing the likelihood of runtime errors and bugs.

  1. Rich Ecosystem

Yew's ecosystem includes libraries and tools to enhance your development experience. For instance, yew-router provides routing capabilities, while yewdux offers state management solutions. The ecosystem is continuously growing, with contributions from the Rust community expanding its capabilities.

  1. WebAssembly Integration

Yew compiles to WebAssembly, enabling your Rust code to run efficiently in the browser. This allows you to leverage Rust's performance benefits in your web applications, providing a smooth and responsive user experience.

3.0.1. Prerequisites

Ensure that you have a Unix-like system (Linux or macOS) for these instructions. If you don't have Rust installed yet, you can use Nix to install and manage Rust, which is useful for creating isolated and reproducible development environments.

  1. Install Nix

First, you need to install Nix, a package manager that allows you to create isolated development environments. 

# Install Nix as a single-user installation
sh <(curl -L https://nixos.org/nix/install) --no-daemon
  1. Set Up the Nix Development Environment

Create a directory for your Rust development environment and navigate into it: 

mkdir rust-shell
cd rust-shell
  1. Download the necessary configuration files:

These files configure your Nix environment to include Rust and related tools. 

wget https://raw.githubusercontent.com/abhi-xyz/nix-shells/master/rust/shell.nix
wget https://raw.githubusercontent.com/abhi-xyz/nix-shells/master/rust/flake.nix
wget https://raw.githubusercontent.com/abhi-xyz/nix-shells/master/rust/flake.lock
wget https://raw.githubusercontent.com/abhi-xyz/nix-shells/master/rust/rust-toolchain.toml
  1. Enter the Nix Development Environment

Use the following command to enter the Nix shell. This environment is isolated, so any software you install or changes you make will not affect your system outside of this shell. 

nix develop
  1. Install Trunk

Once inside the Nix shell, you can use Cargo (Rust's package manager and build system) to install Trunk, a tool for building and bundling Rust applications. 

cargo install trunk
  1. Exiting the Nix Shell

When you exit the Nix shell, any changes or installations made within that environment are no longer accessible. To return to the environment, you must use nix develop again. Uninstalling Nix

If you need to uninstall Nix at any point, you can follow the instructions provided in the Nix manual.

4. Why Recommend Nix?

Nix is highly recommended for managing development environments due to the following benefits:

Isolation: Nix provides isolated environments that ensure changes made in one environment do not impact the global system or other environments. This isolation is particularly valuable in web development, where tools like Trunk can leave services or ports in use even after exiting. In contrast, when you exit a Nix shell, all processes started within it are terminated, freeing up resources such as ports. This prevents conflicts and allows you to reuse ports seamlessly across different sessions.

Reproducibility: With Nix, you can achieve consistent and reproducible development environments. By using configuration files like shell.nix and flake.nix, you ensure that all team members and collaborators work with the same tools and dependencies. This eliminates discrepancies and simplifies sharing development environments. For example, by sharing these configuration files, you can easily provide others with a setup that mirrors your own.

Declarative Configuration: Nix leverages declarative configuration files to define environments. Files like shell.nix and flake.nix describe exactly what the environment should include, making it straightforward to replicate and share setups across different machines and teams. This declarative approach enhances clarity and maintainability.

Flexibility: Nix allows you to specify exact versions of tools and dependencies, reducing the risk of version conflicts and compatibility issues. This precision ensures that all developers are using the same versions, which helps prevent the "works on my machine" problem and facilitates smoother collaboration.

By incorporating Nix into your workflow, you streamline your development process, avoid environment-related issues, and promote consistency across various systems and team members.

5. Creating a New Yew Project

cargo new my-site

cd my-site


[package]
name = "my-site"
version = "0.1.0"
edition = "2021"

[dependencies]
yew = { version = "0.21.0", features = ["csr"] }
yew-router = "0.18.0"


use yew::prelude::*;

struct Model {
    link: ComponentLink<Self>,
    counter: i64,
}

enum Msg {
    Increment,
}

impl Component for Model {
    type Message = Msg;
    type Properties = ();

    fn create(link: ComponentLink<Self>) -> Self {
        Self {
            link,
            counter: 0,
        }
    }

    fn update(&mut self, msg: Self::Message) -> ShouldRender {
        match msg {
            Msg::Increment => {
                self.counter += 1;
                true
            }
        }
    }

    fn view(&self) -> Html {
        html! {
            <div>
                <button onclick=self.link.callback(|_| Msg::Increment)>{ "Increment" }</button>
                <p>{ self.counter }</p>
            </div>
        }
    }
}
